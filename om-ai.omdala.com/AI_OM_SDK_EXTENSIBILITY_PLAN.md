# Om AI SDK Extensibility Plan

Version: 1.0
Status: Draft for DEV handoff

## 1. Purpose

Define how third-party skills, connectors, and extensions may integrate with Om AI later.

## 2. Extensibility Layers

- app intents
- connector plugins
- gateway plugins
- scene templates
- policy templates
- memory import/export

## 3. Extension Rules

- extensions must be policy-aware
- extensions must not bypass proof
- extensions must declare capabilities explicitly
- extensions must support versioning

## 4. Future SDK Goals

- third-party scene packs
- third-party device connectors
- business workflow templates
- safe automation modules
