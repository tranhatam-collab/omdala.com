# OMDALA Product Principles

## Purpose

Define product behavior rules that the implementation must preserve.

## Principles

1. Production first.
2. Clarity over cleverness.
3. Small safe change over broad rewrite.
4. Full file output for implementation work.
5. No mock data unless requested.
6. No silent architectural drift.

## Decision Rules

- If a change affects structure, pause and ask.
- If a change is local, keep it local.
- If a feature is unclear, define the minimal useful version first.

## Next Actions
