# OMDALA Brand Architecture

Version: 1.0
Status: approved master brand direction

## 1. Brand role

OMDALA is the master brand.

It is the global system layer, not a local implementation.

OMDALA should hold:

- platform language
- trust architecture
- intelligence layer
- app logic
- API logic
- product system

## 2. Positioning

Primary:

- OMDALA
- The Operating Layer for Real-World Value

Secondary:

- Activate People, Places, and Intelligence

## 3. Category

Approved category language:

- Human Coordination OS
- Life Resource Operating System

Do not describe OMDALA as:

- social app
- marketplace
- AI tool
- travel platform

## 4. Brand traits

OMDALA must feel:

- calm
- premium
- intelligent
- structured
- trustworthy

## 5. Brand system

```text
OMDALA
  OMDALA Home
  OMDALA Node
  OMDALA Trust
  OMDALA Match
  OMDALA Flow
  OMDALA Host
  OMDALA Mobile
  OMDALA Admin
  OMDALAT
```

## 6. Relationship to OMDALAT

OMDALAT is not a separate conceptual universe.

It is:

- the first city node
- the first living lab
- the first proof layer
- the first density engine

Simple rule:

- `OMDALA` is global
- `OMDALAT` is the first real implementation

## 7. Domain strategy

- `omdala.com` = master homepage
- `app.omdala.com` = product app
- `api.omdala.com` = backend
- `docs.omdala.com` = docs

## 8. UI direction

The master brand should feel:

- premium but not luxurious-for-show
- technological but not noisy
- minimal but not empty
- calm but not weak

## 9. Copy rule

Use:

- clear
- operational
- calm
- premium

Avoid:

- hype
- startup jargon
- fake AI language
- travel language

## 10. Final brand rule

If a future feature or page makes OMDALA feel like a generic app, a local listing board, or a hype startup page, reject it.
